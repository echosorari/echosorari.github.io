re：通过arch Linux的安装学习Linux基本指令（Virtualbox）

本文章写于我是纯正的计算机小白却“不辞辛苦”的试图捣鼓arch Linux时出的各种惨剧，总结了一下安装方法和相关名词解释，文章难免有疏漏，还请包涵。

1.测试网络状态

#ping www.baidu.com

PING(packet Internet Groper)，因特网包探索器。Ping是工作在TCP/IP网络体系结构中应用层的一个服务指令。序来用于测试网络连接量的程序。在win的命令行（cmd）以及Linux的系统中都可以用来进行网络连接状态的确定。
ping的作用主要执行验证远程计算机的连接个数、数据包停留时间、传输接收的数据包数（实现方式略）。
说人话就是检测网络连接状态……

2.测试系统时间

#timedatectl status  //查看系统时间状态（不对的话可以输入）
#timedatectl set-ntp true   //调整arch时间至正确时间

timedatectl命令作为system.d系统和系统管理器的一部分，代替旧的传统的用于基于Linux分布式系统的sysvinit守护进程的date命令（当然，现在的linux体统依旧可以通过date命令查看时间）。

RTC time显示的为硬件时钟的时间。通过观测，arch安装时正常的时间显示的应该为伦敦时间（格林威治标准时间G.M.T），RTC time是通过主板上的特殊电路用于记录时间，不受服务器和操作系统开启关闭影响，也称BIOS本地时间。

Local time初始值来自RTC，由内核维护，一般是RTC+时区值。

status指状态，NTP（Network Time Protocol）是用来指计算机时间同步化的一种协议，使计算机对服务器或者时间源做同质化。NTP主要使用识别机制，检查来对时的信息是否真正来自所宣称的服务器，提高对抗干扰的保护机制。

3.测试存储设备并设置分区

# lsblk
# cfdisk /dev/sda
[Select label type -> gpt -> 自主分区 -> write/yes -> Quit ]

lsblk 列出块设备信息
我们装机的时候使用的时候是默认情况，一般会以树状列出所有块设备。
MAJ:MIN显示主要次要的设备号。
RO表示设备是否只读。
TYPE显示设备是否是磁盘或者像是磁盘上的一个分区。sda，sdb是磁盘，sr是制度存储（rom）。
MOUNTPOINT指出设备挂载点，在安装的时候是没有进行挂载的，后面的步骤会进行挂载。

关于设置分区
此处选择了gpt，GPT的意思是（GUID Partition Table),全局唯一标识磁盘分区表。他是一种使用UEFI启动的磁盘组织模式。(UEFI是什么？UEFI是统一可拓展固件接口，负责POST、联系操作系统、提供链接操作系统与硬件的接口。可以支持2TB以上的硬盘引导操作系统并且快速引导系统或者从休眠状态恢复)。gpt分区的数据引导记录是分开存储的，并且理论上支持无限量分区。
一些分区是必须要有的：
·一个根分区（挂载在根目录）/
·一个EFI系统分区（如果挂载UEFI模式）
·Swap分区（在系统物理内存不够用的时候提供硬盘内存空间供当前运行程序使用的一种分区）

4.格式化分区

#mkfs.fat -F32 /dev/sda1    //EFI分区需要FAT32文件格式
#mkds.ext4 /dev/sda2         //将这个分区格式化成ext4文件系统（这个貌似就是根分区）


#mkswp /dev/sda3 -L Swap  //swap分区的格式化和设置
#swapon /dev/sda3

mkfs就是make filesystem的缩写，用于在特定的分区建立linux文件系统，mkfs实际上是在调用命令。常见的文件系统又ext2（第二代文件拓展系统），ext3，vfat（虚拟文件分配表，主要用于处理厂文件名）等。

dev是设备（device）的英文缩写，这个目录十分重要，因为他包含了所有Linux系统中使用的外部设备，但这里并不放设备的驱动程序，实际上只是访问这些外部设备的端口。我们可以非常方便的去访问这些外部设备，这和访问一个文件，一个目录没有区别。

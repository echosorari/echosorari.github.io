幻想，泡沫与我的一些想法

发表于 2021-05-30

抱着蛮颓废的心情写下的这篇文章吧。毕竟是个人博客一般不会有其他人看到……不过这个博客最初的作用也是用来写随笔以及自己写的一些好玩的小玩意的，所以或许无伤大雅。当然，毕竟也是我不想看到自己再用各种理由搪塞自己无能的事实了。

关于我最近的想法

1.自身状况

从前几个星期开始就一直浑浑噩噩的，头胀，躁动，完全像无头苍蝇一样到处乱撞。蛮想找个人谈谈的，结果悲哀的发现一是找不到该和谁谈自己的现状，二是不知从何说起，估计只能“相顾无言，唯有泪千行” 了。
我一直在问自己“怎么了”，好消息是身体没出任何问题，除了依旧缺乏锻炼之外；坏消息是我也无法判断自己的心理是否出现问题。对自己的所有进行了一次彻底的反问后，使得本身就有些卑微、没有任何自信的我更加感觉自己一无是处了（苦笑），甚至对自己以往的感受和情绪产生了巨大的怀疑。和另一位一直玩的比较好的朋友聊过之后，发现自己的眼界越来越小，越来越低。可悲的是，这种状况持续到现在，我却没有任何办法去解决这个问题。人愈发从本身就并非勤奋之人变得更为好吃懒惰。我不愿意看到这个是“我”这个个体堕落的开端，不然也不会大半夜的试图写下这篇文章来警醒自己了。

2.学习方向的问题

说实话，完全不知道自己在做什么，该做什么。思维的僵化、不愿意去思考问题、因为一件事情比较复杂就直接放弃进行……蛮讨厌现在的自己整个人。从前几周到现在，我对自身抱着极大的厌恶感。
我目前想学的东西或者正在学的有哪些呢，笑死，根本没有正在学的。想学的东西倒是不少：CTF，前端开发，板绘，VOCALOID COVER，编曲，计算机系统，数据结构，西方哲学史，日语N1。这些东西本身就够我喝一壶的了，完全不知道该怎么动手（多么好的一个借口啊，其中许多项目都已经有人为你指引好了方向，结果一句“完全不知道怎么下手”直接掩盖住了）。接下来还是分条谈谈吧。

（1）CTF以及前端开发

网络安全信息竞赛，这个是我接触的相对较早的一个自己比较感兴趣的一个“项目”。毕竟与自身专业相关嘛，多学点也没什么不好。朋友曾经问过我“为什么我们这个信息安全专业的人都想进入校队（网安小组）呢，或者说，为什么他们觉得进入小组是一件水到渠成的、理所当然的事情呢”？当时，我对这个问题也是无言以对。因为他说的很有道理，现在想来，即使和他比较要好的我当时也向这个问题中的想法一样。结果现在遇到了一个比较有趣的事情：我的知识储备根本支撑不起我去参加第十四周（6.6）周末的校赛。曾经和我在同一起跑线的人，如今已有两位达到了可以外出比赛的程度，所以同台竞争，我拿什么去和他们一起竞争？所以，我放弃了校赛。
在做出这个决定之前，好多人还是劝我去打一打尝试一下的。但是我还是决定不去参加校赛了。这个问题先搁置一下。我想说一说我对前端开发的想法。
这个问题曾经和一个学姐讨论过。开发出自己想要的东西的感觉真的很愉快，并且，“开发”更容易让我快速实现经济独立和自由。开发路线那么多为什么选前端呢？三大开发中，我不具备支撑后端开发的强大逻辑（笑死，谁有能力不想当全栈选手呢），无法接受移动端开发这种移动端功能的东西，所以，我非常快速的选择了所谓的“前端”。和我预想的一样，我没有充分的动力去驱动我学习前端，一直搁置到现在还没有办法独立进行前端开发。
比较有意思的是，我所知道的班级内部学习前端的极多。问其原因，大多数的回答都是“因为前端挣钱多啊，后端学的东西又多又累，抽象的很，前端多快乐”。听到这里，心情蛮复杂的。从我来看，前端和后端消耗的精力并没有高低之分，或许有竞争的嫉妒的成分？我个人认为对后端和前端这种极大的偏见是不好的，并且不是所有人都适合学习前端。单纯的抱着“赚钱多”的想法去学习一项东西有没有错误呢，我也不清楚……

（2）板绘，VOCALOID COVER以及编曲

这个没有什么好疑惑的。编曲以及板绘都是因为想写VOCALOID而发展出来的产物，只不过后来，遇到了自己喜欢的唱见，遇见了想写成歌的故事，遇见了遇见本身而被我赋予了新的含义而已。但是，确实对这方面感兴趣，所以动了想要学习的念头。
从初中感兴趣到现在，买了许多相关的书籍，有一搭没一搭的自己看着。这一块着实是我坚持着最久的一块。就算我没有去说真真正正的做出些什么像样的作品，但是我一直在做。有些自我努力自我感动的成分在里面，毕竟学习效率如此低下。但是，骄傲的是，我有在学23333。

我多次询问自己是否还在坚持学习这一方面的初心，索性的是，如今我依旧还是抱着以前的“天真”想法去自己尝试的去写自己感兴趣的曲子，扒着喜欢歌手的谱子。现在坐在电脑屏幕面前的我仍然嘲笑着自己想法的天真。但是我还秉承着这种想法哦，人本身确实蛮奇怪的。

（3）语言类，以及哲学

一个英语没有学好的人，竟然想去学日语。我自己感觉蛮离谱的。手机里有《标准日本语》的电子书，以及日语考级，却连五十音都是背诵了又忘记，一遍遍的背诵一遍遍的忘掉。感谢我这拙劣的脑回路，到如今依旧没有记清楚五十音，也没有继续深入的学习。现在四级考试都要手忙脚乱的去准备。哲学这歌东西是初中一个同学带我入门的，感谢让我打开了新世界的大门，更加的具有思考力和自我反思能力。不过比较麻烦的是如今我对书籍的读取非常慢，甚至已经到了寸步难行的地步。幸好，我没有放弃。

共性，以及戳破的泡泡

在码了两千多字之后，现在是2021.5.30 1:43a.m.耳边依旧是我放松心情用的八音盒音乐。夜深的时候很容易使人焦躁的心情平静下来呢。在各种情绪都压下去之后，只有平静这一种心情充斥在心中。我有感觉我可以了。当然，写这篇文章并不是为了这样说服不完美的自己，而是为了戳破“泡泡”。
（突然感觉下面说话都轻快乐观起来了233）
谈一谈共性吧，做到一半就半途而废了，思维逐渐变得僵化起来，懒。这不是当代大学生的共性嘛（突然冒犯）。认真的说，我一直活在自欺当中。不如说大家都活在自欺当中。很感谢自欺一定程度上保证了我晚上可以拥有一个好的睡眠，可以不用害怕过去的事情。虽然他短暂的失效过。可是，是时候不去拿“体谅父母所以做事情老老实实，不给他们添麻烦”这种当挡箭牌，以“因为懒所以没有做完很正常”当借口，以“因为是山科所以没办法嘛”作为没有机遇的理由了。虚幻的泡沫该消逝了，平静的流淌吧。

13.安装X窗口系统

# pacman -S xorg

xorg就是X窗口系统，xorg是X11的一个实现，而X Window System是一个C/S结构的程序，Xorg只是提供了一个X Server，负责底层的操作当你运行一个程序的时候，这个程序会连接到X server上，由X server接收键盘鼠标输入和负责屏幕输出窗口的移动，窗口标题的样式等等。

# pacman -S xf86-input-synaptics

input-synaptics这里指的是安装笔记本电脑的触控板接口，如果是台式机的话就没有很大的必要了。synaptics是一家公司。主要是以人机界面交互业务为主。这个接触版接口就是他们家开发的……

# pacman -S ttf-dejavu wqy-microhei

安装字体：Dejavu 和 微米黑字体（不安装的话 后面进入桌面环境设置系统语言为简体中文的时候会出现字体显示不全的问题）。

TTF（TrueTypeFont）是Apple公司和Microsoft公司共同推出的字体文件格式，随着windows的流行，已经变成最常用的一种字体文件表示方式。

DejaVu是一套改造自Bitstream Vera的电脑字体，大幅扩充了Unicode所含盖的字符。DejaVu以自由版权发布，因此亦为许多自由软件采用为显示字体。

14.安装桌面环境

常用的两大桌面环境：GNOME和KDE,这个地方我们主要给予了plasma（KDE最新版本的安装）。啥？你问我区别？自行搜索吧兄得。

# pacman -S plasma \\安装plasma

Plasma 是 KDE 4 中一个关键技术 (被誉为 “KDE 的支柱”)，是用户最直观的东西。　Plasma 是“主管”桌面界面的组件：桌面、面板（经常特指任务栏面板）、以及相关元素。然而，Plasma 不仅仅是这些通常部分至界面。“桌面”自最初拥有以来未曾改变很多：典型的桌面包括快捷方式、一个面板、以及当前运行的程序图标。Plasma 旨在改变这些，融入语义学程序元素，并在简化工作流程的同时以可视化的反应把协作技术带至用户的指尖。

# pacman -S konsole \\安装konsdle控制台终端
# pacman -S dolphin \\安装文件管理器
# pacman -S kde-applications \\安装kde套件
# systemctl enable sddm \\启用sddm显示管理器

Simple Desktop Display Manager (SDDM) 是用于 X11 和 wayland 视窗系统的显示管理器（图形登录界面）。SDDM 完全使用 C++11 编写并支持通过 QML 改变主题。对于KDE Plasma 5，KDE 选择了 SDDM 作为 KDE Display Manager 的接替者。

# systemctl enable NetworkManger \\启用网络管理
# pacman -S plasma-nm \\安装图形界面

8.切换系统，进行语言设置，时间设置

# arch-chroot /mnt /bin/bash
# nano /etc/locale.gen

chroot（change root），就是改变程序执行时所参考的根目录的位置。同时chroot可以以限制使用权限（

bin是binary的简写，主要用于放置系统必备的执行文件；bash是Unix shell的一种，是许多Linux发行版的默认Shell（Shell俗称壳，是指为使用者提供操作界面的软件（命令解析器），他接收用户命令，然后调用相应的应用程序）。

用nano指令打开时区设置后对”en_US.UTF-8 UTF-8 ；zh_CN.UTF-8 UTF-8“进行反注释（删掉前面的#），退出保存。

# locale-gen
# echo LANG =en_US.UTF-8 > /etc/locale.conf

locale是软件正在进行时的语言环境，是根据计算机用户使用的语言、所在国家以及地域、当地文化传统所定义的一个语言环境。它包括语言、地域、以及字符集。locale-gen是编译本地定义文件的一个列表。

echo命令用于字符串的输出，可以打印普通字符串、显示转义字符、显示变量、显示换行风，此处是用来查看时间的正确性。

conf在这里是存放于etc目录下locale文件的拓展名，这个知道就行了，不用过多了了解（）。

“>”是重定向的意思，使用>会直接覆盖掉后面的文件。与之相似的>>则是再文件后方追加的意思，可以在文件中追加内容。

LANG是一种变量，用于定义Linux系统的主语系环境，默认值为安装Linux时候选择的语言。格式一般为“zh_CN.UTF-8”之类的，时区时间格式是固定的，可以自行搜索。

# ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime
# hwclock --systohc --utc //设置硬件时间

ln指令（link files）的功能是为某一个文件在另一个位置建立同步的连接。当我们需要在不同的位置存放同一个文件的时候，一般就在一个固定目录下放置该文件并用ln加上link，这样做可以节省非必要的磁盘空间。
ln -sf的含义具有指向性，比如以上就是将/etc/localtime指向/usr/share/zoneinfo……值得注意的是，我这两个目录之间有一个极其微小的空格（估计有人没看见），将前后的分开代表后方指向前方。

hwclock命令可以用来显示/设置硬件时钟命令。在Linux系统中有硬件时钟和系统时钟两种时钟。这个就是用来设置硬件时钟的。hwclock -systohc则是表示将硬件时钟调整为与目前系统时钟时间一致。

UTC是指协调世界时，又称为世界统一时间、国际协调时间。其他时间都是通过这玩意换算过来的……

GRUB（GRand Unified Bootloader）是多操作系统启动程序，它允许用户可以在计算机内同时拥有多个操作系统，并且在计算机启动时选择希望运行的操作系统。这一步引导装载程序实际上是引出更高级的功能，以允许用户装载并将控制权交给特定的操作系统。

4.5.挂载分区

（草率了……上次没讲完）

挂载分区的时候注意按顺序挂载，先挂在根分区到/mnt
# mount /dev/sda2 /mnt
# mkdir -p /mnt/boot/EFI
# mount /dev/sda1 /mnt/boot/EFI

mount（mnt是其缩写）为挂载的意思，它可以将分区挂接再Linux的文件夹下从而使得分区和目录联系起来，从而使得访问文件夹的时候就相当于访问了分区，没有挂载分区的Linux系统重启后会直接开启装机系统界面……不知道什么鬼……

5.修改镜像源

# nano /etc/pacman.d/mirrorlist
# Server = https://mirrors.aliyun.com/archlinux/$repo/os/$arch  //配置阿里云镜像。
# Server = https://mirrors.tuna.tsinghua.edu.cn/archlinux/$repo/os/$arch
# Server =  https://mirrors.bfsu.edu.cn/archlinux/$repo/os/$arch

nano 是一个字符终端的文本编辑器，有点像DOS下的editor程序，相对vim而言要简单的多，更适合Linux初学者。
nano命令课题打开指定的文件进行编辑，上面所展示的就是对arch Linux的镜像源的编辑过程。编辑时找到##China（中国国内的镜像源）并调节到前方，从而加快从镜像中获取软件包等的速度。当然后面还有许多选项，这里只是单纯使用了他的默认模式，需要的可以自行搜索。

vim是从vi发展出来的一个文本编辑器。代码补完、编译以及跳转等方便编程的功能特别丰富，适合资深程序员使用（当然想我这种100%小白就直接用了nano，相对更加的方便快捷）。vim共三种模式，分别为命令模式（Command mode），输入模式（Insert mode）和底线命令模式（Last line mode），再次不多做赘述。

pacman是一个软件包管理器，作为arch Linux的一部分，是arch Linux的一大亮点（过于舒适.jpg）。他将简单的二进制包格式和易用的构建系统结合了起来。不管软件包是官方的arch库还是用户自己创建，Pacman都可以轻松管理。
pacman通过和主服务器同步软件包列表进行系统更新，这种服务器/客户端模式模式可以使用一条指令就能下载安装软件包（我愿称之为懒人快乐包）。
下面提供几条“基本”指令：
·同步：
pacman -Syy
也可以使用一句命令同时进行同步软件库并更新系统到最新状态
pacman -Syu
·安装软件包
安装或者升级单个软件包，或者一列软件包（包含依赖包），使用如下命令：
pacman -S package_name1 package_name2
·有时候在不同的软件仓库中，一个软件包有多个版本（比如extra和testing）。你可以选择一个来安装：
pacman -S extra/package_name
pacman -S testing/package_name
你也可以在一个命令里同步包数据库并且安装一个软件包：
pacman -Sy package_name
·卸载软件包
·删除单个软件包，保留其全部已经安装的依赖关系
pacman -R package_name
·删除指定软件包，及其所有没有被其他已安装软件包使用的依赖关系：
pacman -Rs package_name
·指定根目录
pacman -r /mnt
大写R删除 ，小写r指定根目录
·包数据库查询
·可以使用 -Q 标志搜索和查询本地包数据库。详情参见
pacman -Q –help
·可以使用-S 标志搜索和查询远程同步的包数据库。详情参见
pacman -S –help
·其它
·下载包而不安装它：
pacman -Sw package_name
·安装一个本地包（不从源里）：
pacman -U /path/to/package/package_name-version.pkg.tar.gz
·完全清理包缓存(/var/cache/pacman/pkg)：
pacman -Scc

6.开始安装基本操作系统

（2020和2019以及以前的操作系统安装指令有所修改，不太清楚为啥）

# pacstrap -i /mnt base base-devel linux linux-firmware

pac是package（软件包），strap是用袋子捆扎，所以pacstrap是打包好的软件，属于arch系统底下一个脚本文件的名称，用于安装arch Linux系统。

base包是基础系统，基本上装arch这个包组都要装的，而base-devel（devel = develop，咱也不知道是谁搞出来这么恶心的缩写）里的是一些常用的开发工具，如果不是程序员，大部分base-devel里的包一般是用不到的，当然如果需要编译安装某些软件，就会用到其中的一些编译工具，比如automake，cmake之类的。

linux-firmware（Linux系统固件），很多硬件需要自己的固件才能运行，这种东西类似于硬件自己的操作系统（因为固件自己的储存器用起来有亿点难度），用于进行顾自安的安装。

7.配置fstab

# cat /mnt/etc/fstab

fstab负责配置对已经划分并挂载的分区的开机自动挂载，从而达到将磁盘分区挂载再目录树的具体目录上读写操作的目的。genfstab好像是用来生成fstab的工具。

本文章写于我是纯正的计算机小白却“不辞辛苦”的试图捣鼓arch Linux时出的各种惨剧，总结了一下安装方法和相关名词解释，文章难免有疏漏，还请包涵。

1.测试网络状态

#ping www.baidu.com

PING(packet Internet Groper)，因特网包探索器。Ping是工作在TCP/IP网络体系结构中应用层的一个服务指令。序来用于测试网络连接量的程序。在win的命令行（cmd）以及Linux的系统中都可以用来进行网络连接状态的确定。
ping的作用主要执行验证远程计算机的连接个数、数据包停留时间、传输接收的数据包数（实现方式略）。
说人话就是检测网络连接状态……

2.测试系统时间

#timedatectl status  //查看系统时间状态（不对的话可以输入）
#timedatectl set-ntp true   //调整arch时间至正确时间

timedatectl命令作为system.d系统和系统管理器的一部分，代替旧的传统的用于基于Linux分布式系统的sysvinit守护进程的date命令（当然，现在的linux体统依旧可以通过date命令查看时间）。

RTC time显示的为硬件时钟的时间。通过观测，arch安装时正常的时间显示的应该为伦敦时间（格林威治标准时间G.M.T），RTC time是通过主板上的特殊电路用于记录时间，不受服务器和操作系统开启关闭影响，也称BIOS本地时间。

Local time初始值来自RTC，由内核维护，一般是RTC+时区值。

status指状态，NTP（Network Time Protocol）是用来指计算机时间同步化的一种协议，使计算机对服务器或者时间源做同质化。NTP主要使用识别机制，检查来对时的信息是否真正来自所宣称的服务器，提高对抗干扰的保护机制。

3.测试存储设备并设置分区

# lsblk
# cfdisk /dev/sda
[Select label type -> gpt -> 自主分区 -> write/yes -> Quit ]

lsblk 列出块设备信息
我们装机的时候使用的时候是默认情况，一般会以树状列出所有块设备。
MAJ:MIN显示主要次要的设备号。
RO表示设备是否只读。
TYPE显示设备是否是磁盘或者像是磁盘上的一个分区。sda，sdb是磁盘，sr是制度存储（rom）。
MOUNTPOINT指出设备挂载点，在安装的时候是没有进行挂载的，后面的步骤会进行挂载。

关于设置分区
此处选择了gpt，GPT的意思是（GUID Partition Table),全局唯一标识磁盘分区表。他是一种使用UEFI启动的磁盘组织模式。(UEFI是什么？UEFI是统一可拓展固件接口，负责POST、联系操作系统、提供链接操作系统与硬件的接口。可以支持2TB以上的硬盘引导操作系统并且快速引导系统或者从休眠状态恢复)。gpt分区的数据引导记录是分开存储的，并且理论上支持无限量分区。
一些分区是必须要有的：
·一个根分区（挂载在根目录）/
·一个EFI系统分区（如果挂载UEFI模式）
·Swap分区（在系统物理内存不够用的时候提供硬盘内存空间供当前运行程序使用的一种分区）

4.格式化分区

#mkfs.fat -F32 /dev/sda1    //EFI分区需要FAT32文件格式
#mkds.ext4 /dev/sda2         //将这个分区格式化成ext4文件系统（这个貌似就是根分区）


#mkswp /dev/sda3 -L Swap  //swap分区的格式化和设置
#swapon /dev/sda3

mkfs就是make filesystem的缩写，用于在特定的分区建立linux文件系统，mkfs实际上是在调用命令。常见的文件系统又ext2（第二代文件拓展系统），ext3，vfat（虚拟文件分配表，主要用于处理厂文件名）等。

dev是设备（device）的英文缩写，这个目录十分重要，因为他包含了所有Linux系统中使用的外部设备，但这里并不放设备的驱动程序，实际上只是访问这些外部设备的端口。我们可以非常方便的去访问这些外部设备，这和访问一个文件，一个目录没有区别。
